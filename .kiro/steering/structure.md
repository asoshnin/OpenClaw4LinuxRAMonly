# Project Structure

## Directory Layout

```
OpenClaw4LinuxRAMonly/
├── AGENTS.md                        ← AI agent directives (universal standard)
├── README.md                        ← Project overview and quick start
├── CHANGELOG.md                     ← Version history
├── REGISTRY.md                      ← AUTO-GENERATED by librarian_ctl.py — do not edit manually
├── .gitignore
│
├── .kiro/
│   └── steering/                    ← Kiro steering files (AI context)
│       ├── product.md
│       ├── tech.md
│       ├── structure.md
│       ├── security-hitl.md
│       └── sprint-workflow.md
│
├── openclaw_skills/
│   ├── architect/
│   │   ├── architect_tools.py       ← Discovery, HITL, deploy, teardown CLI
│   │   └── SKILL.md                 ← Agent skill definition for Architect
│   └── librarian/
│       ├── librarian_ctl.py         ← DB init, bootstrap, registry generation
│       ├── migrate_db.py            ← Schema migrations
│       ├── vector_archive.py        ← sqlite-vec: init + Faint Path search
│       └── safety_engine.py         ← Hybrid distillation (Ollama + Gemini)
│
├── docs/                            ← Technical documentation
│   └── YYYY-MM-DD__HH-MM_*.md      ← Timestamped state snapshots
│
├── _Development/
│   └── OpenClaw/
│       ├── YYYY-MM-DD_backlog.md    ← Live sprint backlog
│       ├── Sprint_N_Name/           ← Per-sprint spec directories
│       │   ├── requirements.md
│       │   ├── design.md
│       │   └── tasks.md
│       └── AgenticFactoryOnClaw.md  ← System architecture document
│
├── workspace/                       ← GITIGNORED: runtime state
└── database/                        ← GITIGNORED: factory.db location
```

## Key Patterns

- **CLI tools** use `argparse` subcommands. Entry point is always `if __name__ == "__main__"`.
- **DB path** is always passed as a positional CLI argument — never hardcoded.
- **Sprint specs** follow the Kiro 3-file pattern: `requirements.md`, `design.md`, `tasks.md` inside `_Development/OpenClaw/Sprint_N_Name/`.
- **Docs** use timestamp prefix format: `YYYY-MM-DD__HH-MM_description.md`
- **REGISTRY.md** is regenerated by running: `python3 librarian_ctl.py refresh-registry <db_path> <registry_path>`
