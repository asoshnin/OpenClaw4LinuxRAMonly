## Project Identity

**OpenClaw** is a hardened, self-evolving agentic operating system for Linux x86_64. It is not a web app or a microservice — it is a **local-first, air-gapped, RAM-only agentic runtime** designed for Epistemic Sovereignty.

- **Navigator (human)**: Alexey. All deployment decisions require his explicit approval.
- **Hardware target**: ThinkPad W540 (x86_64, Linux, CPU-only inference)
- **Primary DB**: `factory.db` (SQLite WAL mode) — never touch it directly; always use `librarian_ctl.py`.
- **Sensitive inference**: Local only via Ollama. Cloud (Gemini) is used ONLY for non-sensitive log distillation.

## Critical Safety Rules (NEVER VIOLATE)

1. **HITL Gate is sacred.** Any pipeline deployment MUST go through `deploy_pipeline_with_ui()`. Never bypass the tkinter GUI popup or the Burn-on-Read token. Do not generate or validate tokens manually.
2. **Airlock enforcement.** All file operations must be scoped within the workspace boundary using `os.path.realpath()`. Never suggest code that writes outside the workspace.
3. **System agents are protected.** Agents with `is_system=1` in `factory.db` must NEVER be torn down or modified.
4. **Context Guard.** Logs passed to any LLM must be truncated at 12,000 characters. Do not suggest removing this limit.
5. **Epistemic Scrubber.** All data going into the Vector Archive must pass through the IPI-stripping safety prompt in `safety_engine.py`. Do not bypass it.
6. **No cloud for sensitive data.** Never suggest sending agent profiles, pipeline configs, or audit logs containing PII/sensitive ops to external APIs.

## Code Conventions

- **Language**: Python 3.10+. No async unless explicitly requested.
- **CLI pattern**: All tools expose a CLI via `argparse` subcommands (see `librarian_ctl.py` and `architect_tools.py` as canonical examples).
- **DB access**: Always use parameterized queries. WAL mode is already set; do not change journal mode.
- **Error handling**: Use explicit `try/except` with logged errors. Never swallow exceptions silently.
- **Logging**: Use Python's `logging` module. Log level `INFO` for normal ops, `WARNING` for HITL events, `ERROR` for failures.
- **No external dependencies** beyond: `sqlite-vec`, `pyyaml`, `ollama` (Python client), `google-generativeai`, `tkinter` (stdlib).

## What NOT To Do

- Do NOT suggest Docker, Kubernetes, or cloud deployment for core runtime — this is intentionally air-gapped.
- Do NOT introduce ORM layers (SQLAlchemy etc.) — raw `sqlite3` is intentional for minimal deps.
- Do NOT add async/await patterns unless explicitly asked — synchronous execution is a safety design choice.
- Do NOT modify `REGISTRY.md` by hand — it is auto-generated by `librarian_ctl.py refresh-registry`.
- Do NOT suggest storing secrets or API keys in code — use environment variables only.
