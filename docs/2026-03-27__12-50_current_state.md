## 1. System Overview

The **Agentic Factory** is a self-evolving AI infrastructure built on the OpenClaw framework for Linux. Its core purpose is to serve as a hardened, Human-in-the-Loop (HITL)-governed platform on which the human Navigator (Alexey) can autonomously design, deploy, manage, and decommission AI pipelines.

The system adheres to an **Epistemic Sovereignty** philosophy: it is explicitly designed *not* to act autonomously on high-risk operations. Every destructive or sensitive action is gated behind cryptographically enforced human approval.

> **Important:** The repository contains two distinct codebases. The `src/` directory contains a legacy Letta.ai/macOS implementation that is **strictly isolated and must not be modified or imported**. The canonical, active implementation resides exclusively in `openclaw_skills/`.

## 3. Completed Sprints & Implemented Features

### Sprint 1 — The Librarian (`librarian_ctl.py`)

**Purpose:** Persistent state management, Airlock security, and registry generation.

#### 3.1.1 `validate_path(target_path: str) -> str`
The system's primary security primitive. Enforces the Airlock boundary.

```python
def validate_path(target_path: str) -> str:
    base_dir = os.path.realpath("/home/alexey/openclaw-inbox/workspace/")
    target_abs = os.path.realpath(target_path)
    if not target_abs.startswith(base_dir):
        raise PermissionError(f"Airlock Breach: {target_abs} is outside {base_dir}")
    return target_abs
```
- Uses `os.path.realpath()` to resolve symlinks before comparison — prevents all symlink traversal attacks.
- Raises `PermissionError` immediately on violation.

#### 3.1.2 `init_db(db_path: str) -> None`
Initializes `factory.db` with full schema and durability settings.

**Database Schema:**
```sql
-- Durability
PRAGMA journal_mode=WAL;
PRAGMA synchronous=NORMAL;  -- Paired with WAL for optimal durability/performance on W540

-- Core tables
CREATE TABLE IF NOT EXISTS agents (
    agent_id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    version TEXT DEFAULT '1.0',
    persona_hash TEXT,
    state_blob JSON,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS pipelines (
    pipeline_id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    topology_json JSON,
    status TEXT CHECK(status IN ('active', 'archived', 'deprecated'))
);

CREATE TABLE IF NOT EXISTS audit_logs (
    log_id INTEGER PRIMARY KEY AUTOINCREMENT,
    agent_id TEXT,
    pipeline_id TEXT,
    action TEXT,
    rationale TEXT,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY(agent_id) REFERENCES agents(agent_id),
    FOREIGN KEY(pipeline_id) REFERENCES pipelines(pipeline_id)
);
```

Seeds the DB with the two core system agents and the root pipeline using `INSERT OR IGNORE` (idempotent).

#### 3.1.4 `generate_registry(db_path: str, output_md_path: str) -> None`
Generates a human-readable Obsidian-ready Markdown registry from the live DB state.
- **Atomic write pattern:** writes to `output_md_path + ".tmp"` first, then calls `os.replace()` — safe against IDE file watcher race conditions.
- Output format: YAML frontmatter + Markdown body sections for agents and pipelines.
- **Airlock note:** Both `db_path` AND `output_md_path` pass through `validate_path()` before use. The output registry file is also Airlock-protected, not just the DB path.

### Sprint 2 — The Architect & HITL Gates (`architect_tools.py`)

**Purpose:** Discovery, secure deployment, and lifecycle orchestration with Burn-on-Read HITL.

#### 3.2.1 `validate_path()` — Architect copy
Identical Airlock implementation to Librarian (ported, not imported — isolation constraint).

#### 3.2.2 `search_factory(db_path, query_type, filter_val=None) -> list`
Read-only SQLite discovery tool. Supported `query_type` values: `agents`, `pipelines`, `audit` (alias for `audit_logs`). Accepts optional `filter_val` for single-entity lookup. Uses `conn.row_factory = sqlite3.Row` for dict-like access.

> **⚠️ Audit filter behaviour (DELTA-09):** For `query_type='audit'`, `filter_val` is matched against BOTH `agent_id` AND `pipeline_id` via `OR`. A single `filter_val='kimi-orch-01'` returns all audit records where either column matches — broader than a strict agent-only or pipeline-only filter. Callers relying on narrow filtering should post-filter results in Python.

#### 3.2.3 `get_agent_persona(db_path, agent_id) -> dict`
Convenience wrapper over `search_factory` for single-agent lookup.

#### 3.2.4 Token System — Burn-on-Read HITL

**`generate_token() -> str`**
- Generates a `uuid.uuid4()` string.
- Writes to `workspace/.hitl_token` using `os.open()` with flags `O_WRONLY | O_CREAT | O_TRUNC` and mode `0o600` (user-only read/write).
- Returns the token string for display.

**`validate_token(provided_token: str) -> bool`**
- Reads stored token from `.hitl_token`.
- **Immediately calls `os.remove(TOKEN_FILE)` — the file is destroyed at the moment of read, before the comparison.** This prevents replay attacks even if comparison fails.
- Returns `True` only if `stored_token == provided_token.strip()`.

#### 3.2.5 `deploy_pipeline(db_path, pipeline_id, pipeline_name, topology_json, approval_token) -> str`
- Calls `validate_token(approval_token)` first — raises `PermissionError` if invalid/expired.
- Inserts into `pipelines` table.
- Inserts into `audit_logs` with action `DEPLOY_PIPELINE`.

#### 3.2.6 `request_ui_approval(prompt_text: str) -> bool`
Native Linux GUI popup using `tkinter.messagebox.askyesno()`.
- Graceful headless fallback: if `tkinter` is unavailable (`TK_AVAILABLE = False`), falls back to a terminal `input("Approve? (yes/no): ")` prompt.
- Window is always rendered `topmost` to prevent it hiding behind other windows.

#### 3.2.7 `deploy_pipeline_with_ui(db_path, pipeline_id, pipeline_name, topology_json) -> str`
The primary tool the AI Architect agent calls. High-level wrapper that:
1. Calls `request_ui_approval()` — blocks execution, shows GUI popup to Navigator.
2. On "No": raises `PermissionError("Deployment rejected by human Navigator via UI.")`.
3. On "Yes": internally calls `generate_token()` and immediately feeds it into `deploy_pipeline()`.

This design means the agent **never sees or handles the HITL token**. The human's "Yes" click _is_ the token.

**CLI Interface:**
```bash
python3 architect_tools.py gen-token
python3 architect_tools.py deploy <db_path> <pipeline_id> <name> <topology_json> <token>
python3 architect_tools.py teardown <db_path> <pipeline_id>
```

### Sprint 3 — Hybrid Vector Archive (`vector_archive.py`, `safety_engine.py`)

**Purpose:** Semantic memory layer for "Faint Path" retrieval and Safety Distillation.

#### 3.3.1 `init_vector_db(db_path: str) -> None` (`vector_archive.py`)
- Connects to SQLite, loads `sqlite-vec` extension via `sqlite_vec.load(conn)`.
- Creates two tables:

```sql
CREATE VIRTUAL TABLE IF NOT EXISTS vec_passages USING vec0(
    passage_id INTEGER PRIMARY KEY,
    embedding FLOAT[768]      -- nomic-embed-text output dimension
);

CREATE TABLE IF NOT EXISTS distilled_memory (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    raw_source_id TEXT,
    content_json JSON,         -- {"facts": [...], "scrubbed_log": "..."}
    is_sensitive BOOLEAN,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.3.2 `find_faint_paths(db_path, query_text, limit=5) -> list[dict]` (`vector_archive.py`)
- Instantiates `SafetyDistillationEngine` and calls `_get_embedding(query_text)`.
- Loads `sqlite-vec` and executes sqlite-vec compliant KNN query:
```sql
SELECT v.passage_id, v.distance, m.raw_source_id, m.content_json, m.is_sensitive, m.timestamp
FROM vec_passages v
JOIN distilled_memory m ON m.id = v.passage_id
WHERE v.embedding MATCH ? AND k = ?     -- k constraint required for JOIN pushdown
ORDER BY v.distance
```
- Returns list of dicts with `content_json` auto-parsed from JSON string.

#### 3.3.3 `truncate_for_distillation(text, limit=12000) -> str` (`safety_engine.py`)
Middle-truncate guardrail for context window protection on W540 hardware.
- If `len(text) > 12000`: keeps first 6,000 and last 6,000 chars, inserts `"...[TRUNCATED FOR RESILIENCE]..."` separator.
- Called at the entry point of both `_distill_local` and `_distill_cloud` before any prompt construction.

#### 3.3.4 `SafetyDistillationEngine` class (`safety_engine.py`)

| Attribute | Value |
|---|---|
| `ollama_url` | `http://127.0.0.1:11434` |
| `local_model` | `nn-tsuzu/lfm2.5-1.2b-instruct` |
| `cloud_model` | `gemini-3.1-flash-lite-preview` |
| `embed_model` | `nomic-embed-text` |

**`_get_embedding(text) -> list[float]`**
- HTTP POST to `{ollama_url}/api/embeddings` with `{"model": "nomic-embed-text", "prompt": text}`.
- `timeout=30.0` enforced on all urllib calls.
- Returns 768-dimensional float vector.

**`_distill_local(raw_log) -> dict`**
- Applies `truncate_for_distillation(raw_log)` first.
- HTTP POST to `{ollama_url}/api/generate` with `"format": "json"` enforced.
- Uses the [DES-14] Epistemic Scrubber prompt.
- Parses `result["response"]` as JSON; fallback returns `{"facts": [], "scrubbed_log": raw_text}`.

**`_distill_cloud(raw_log) -> dict`**
- Applies `truncate_for_distillation(raw_log)` first.
- Reads `GEMINI_API_KEY` from environment — raises `ValueError` if missing.
- HTTP POST to `https://generativelanguage.googleapis.com/v1beta/models/{cloud_model}:generateContent?key={api_key}`.
- Sets `"generationConfig": {"response_mime_type": "application/json"}` for forced JSON output.

**`distill_safety(raw_log, is_sensitive=True) -> dict`**  
The routing entry point:
- `is_sensitive=True` → `_distill_local()` (air-gapped, CPU-only, W540-optimized)
- `is_sensitive=False` → `_distill_cloud()` (Gemini cloud)

**`archive_log(db_path, raw_source_id, raw_log, is_sensitive=True) -> int`**
Full end-to-end pipeline:
1. `distill_safety(raw_log, is_sensitive)` → `distilled_data` dict
2. `_get_embedding(json.dumps(distilled_data))` → 768-dim vector (always local, even for cloud-distilled content)
3. Checks `_SQLITE_VEC_AVAILABLE` — raises `RuntimeError` with install instructions if `sqlite-vec` is missing
4. `INSERT INTO distilled_memory` → gets `new_id`
5. `INSERT INTO vec_passages (passage_id=new_id, embedding=...)`
6. Returns `new_id` (passage_id)

### Sprint 3.5 — Lifecycle & Resilience (`migrate_db.py`, `architect_tools.py`)

**Purpose:** Prevent zombie agents, enforce garbage collection, and secure the engine against hardware failures.

#### 3.4.1 `migrate_database(db_path: str)` (`migrate_db.py`)
Standalone migration runner. Applies three schema changes:

```sql
-- 1. System protection flag (idempotent via try/except)
ALTER TABLE agents ADD COLUMN is_system BOOLEAN DEFAULT 0;

-- 2. Dependency tracking
CREATE TABLE IF NOT EXISTS pipeline_agents (
    pipeline_id TEXT,
    agent_id TEXT,
    PRIMARY KEY (pipeline_id, agent_id),
    FOREIGN KEY(pipeline_id) REFERENCES pipelines(pipeline_id) ON DELETE CASCADE,
    FOREIGN KEY(agent_id) REFERENCES agents(agent_id) ON DELETE CASCADE
);

-- 3. Protect core agents
UPDATE agents SET is_system = 1 WHERE agent_id IN ('kimi-orch-01', 'lib-keeper-01');
```

`ALTER TABLE` is wrapped in `try/except sqlite3.OperationalError` to detect and skip the "duplicate column name" error — making the migration idempotent.

**CLI:**
```bash
python3 migrate_db.py <db_path>
```

#### 3.4.2 `teardown_pipeline(db_path, pipeline_id) -> str` (`architect_tools.py`)
Implements "Check-Before-Kill" garbage collection.

```
Algorithm:
1. SELECT agent_id FROM pipeline_agents WHERE pipeline_id = ?  (snapshot before cascade)
2. DELETE FROM pipelines WHERE pipeline_id = ?                  (triggers ON DELETE CASCADE → pipeline_agents)
3. For each agent_id in snapshot:
   a. If is_system == 1 → SKIP (system-protected)
   b. If COUNT(*) FROM pipeline_agents WHERE agent_id = ? > 0 → SKIP (shared)
   c. Else:
      - DELETE FROM agents WHERE agent_id = ?
      - validate_path(workspace/{agent_id}.md) → os.remove() if exists
4. INSERT INTO audit_logs (action='TEARDOWN', rationale='Pipeline Decommissioned safely.')
5. Return summary string: "Deleted: [...]. Skipped: [...]."
```

> **⚠️ Known behaviours (DELTA-12, NOTE-08):**
> - File deletion exceptions in step 3c are silently caught and reported only via the summary string (e.g., `"agent-id (db only, file err)"`). A `PermissionError` on a read-only profile file will not abort the teardown — the DB record is still deleted.
> - The `TEARDOWN` audit log entry in step 4 is written unconditionally, even if `pipeline_id` did not exist in the DB (zero rows deleted). This can produce orphan audit records for non-existent pipelines.

### Sprint 6 — Core Intelligence & Resilience (`router.py`, `kb.py`, `self_healing.py`)

**Purpose:** HITL-guarded LLM routing, supervised knowledge base, self-healing JSON parsing, and scoped epistemic scrubber.

#### 3.6.1 Dynamic LLM Router (`router.py`)
`route_inference(task_text, is_sensitive, min_model_tier, db_path) -> str` enforces the 6-row routing matrix:
- `is_sensitive=True` + `min_model_tier="cloud"` → **unconditional `PermissionError`** with sentinel `[SYS-HALT: HITL REQUIRED - SENSITIVE CLOUD ROUTING]`. Cloud API never called.
- Ollama availability ping (3s timeout) before every local call — no silent cloud fallback if Ollama is down.
- Every routing outcome (`ROUTE_LOCAL`, `ROUTE_CLOUD`, `ROUTE_LOCAL_FAIL`, `ROUTING_HALT`) is written to `audit_logs` unconditionally.
- CLI defaults: `--sensitive --tier local` (safest possible).

#### 3.6.2 Static Knowledge Base & Reflection Queue (`kb.py`, `knowledge_base.json`)
- `knowledge_base.json` is a **committed static file** with `security_rules`, `capability_boundaries`, `epistemic_invariants`. Never vectorized.
- `run_agent()` injects KB as the first prompt block (KB → identity → memory → task) so rules cannot be overridden by crafted task text.
- Agents may call `submit_kb_proposal()` to queue rule changes — status `pending` in `proposed_kb_updates` table.
- Only `approve_kb_proposal()` (requires HITL burn-on-read token) can apply changes. Atomic write via `os.replace()`.
- New DB migration: `proposed_kb_updates` table (8 columns including HITL-guarded `status` CHECK constraint).

#### 3.6.3 Self-Healing JSON Parser (`librarian/self_healing.py`)
`parse_json_with_retry(raw_text, model_call_fn, max_retries=3)` — circuit breaker:
- Each `JSONDecodeError` → call `model_call_fn` with repair prompt, increment counter.
- After `max_retries` attempts: raises `RuntimeError("JSON parse circuit breaker tripped after N retries")`. **Never returns a degraded fallback dict.**
- `_distill_local()` in `safety_engine.py` now uses this instead of the former silent `except json.JSONDecodeError` fallback.
- `_call_ollama()` extracted as a private helper shared between distillation and repair calls.

#### 3.6.4 Scoped Epistemic Scrubber (`safety_engine.py`)
`archive_log()` gains `source_type: str = "external"` parameter:
- `source_type="external"` (default): calls `distill_safety()` → embedding. Existing behaviour preserved.
- `source_type="internal"`: skips `distill_safety()` entirely, embeds raw log directly. Reduces per-archive LLM calls from **2 → 1** for internal system audit content on CPU-bound W540.
- New DB migration: `source_type TEXT DEFAULT 'external'` column on `distilled_memory` (skipped gracefully if vector tables not yet initialized).

### Sprint 7 — Obsidian Bidirectional Sync (`obsidian_bridge.py`)

**Purpose:** Connect OpenClaw to an Obsidian knowledge base via the local REST API while preserving epistemic sovereignty and avoiding path traversal or PII leakage to the cloud. **94/94 tests pass.**

#### 3.7.1 Local REST API Client (`obsidian_bridge.py`)
- Standard library (`urllib.request`) implementation connecting to `http://127.0.0.1:27123`.
- **Enforced Security Invariants:**
  - `base_url` MUST resolve to loopback (`localhost`, `127.0.0.1`). Remote endpoints raise `ValueError` at construction.
  - `OBSIDIAN_API_KEY` is completely mandatory.
  - Dual path validation preventing directory traversal (`../`) and absolute paths (`/etc/passwd`).
  - Strict compliance with API contract: `PATCH` for appending, `PUT` for writing.

#### 3.7.2 Vault Writing (`architect_tools.py`)
`write_agent_result_to_vault(..., is_sensitive: bool)`
- **Sensitivity Gate:** If `is_sensitive=True`, the write is aborted completely (logging `VAULT_WRITE_REFUSED_SENSITIVE`). Prevents sensitive agent logic from leaking to Obsidian Sync.
- **Context Guard:** All note contents are truncated to a maximum of 12,000 characters before vault insertion.
- Graceful degradation on API absence (logs `VAULT_WRITE_SKIPPED` instead of crashing).
- New CLI: `architect_tools.py write-to-vault`.

#### 3.7.3 Vault Ingestion (`librarian_ctl.py`)
`ingest_vault_note(..., is_sensitive: bool)`
- Fetches raw Markdown from vault and sends it through `SafetyDistillationEngine`.
- **IPI Size Gate:** Notes over `VAULT_INGEST_MAX_BYTES` (50,000) are blocked before hitting context-limited LLMs.
- Forces `source_type="external"`, guaranteeing all ingested vault data is heavily scrubbed.
- New CLI: `librarian_ctl.py ingest-vault-note`.

#### 3.7.4 Zero-Dependency Testing
- Complete mocking array built into `conftest.py` injecting mock `OBSIDIAN_API_KEY`. Real Obsidian instance never required to run the test suite.

## 4. Security Architecture Summary

| Mechanism | Implementation | Bypass Risk |
|---|---|---|
| **Airlock Path Validation** | `os.path.realpath()` + `os.sep` suffix guard on `WORKSPACE_ROOT` from `config.py` | None — symlinks resolved; prefix-collision blocked |
| **Burn-on-Read Token** | `os.remove(TOKEN_FILE)` before comparison result is used | None — file destroyed even if comparison fails |
| **Token File Permissions** | `os.open(..., mode=0o600)` — user-only read/write | None — OS enforced |
| **GUI HITL Gate** | `tkinter.messagebox.askyesno()` blocks agent execution | Headless fallback to terminal prompt |
| **Agent Never Sees Token** | `deploy_pipeline_with_ui` generates and consumes token internally | Agent cannot pass a pre-generated token |
| **HITL-Guarded Cloud Routing** | `route_inference()` raises `PermissionError` (sentinel) when `sensitive + cloud` — cloud never called | None — checked as first condition in router |
| **No Silent Cloud Fallback** | `ROUTE_LOCAL_FAIL` → `RuntimeError`; no retry to cloud | None — RuntimeError propagates to caller |
| **KB Self-Modification Block** | `approve_kb_proposal()` requires HITL burn-on-read token; agents can only `submit_kb_proposal()` | None — token consumed before any write |
| **JSON Circuit Breaker** | `parse_json_with_retry()` raises after `max_retries` — never returns degraded dict | None — explicit RuntimeError |
| **System Agent Protection** | `is_system=1` flag prevents teardown of `kimi-orch-01` and `lib-keeper-01` | — |
| **Cascade-Safe Teardown** | Snapshot agents _before_ pipeline deletion, then check `pipeline_agents` _after_ | — |
| **Network Timeouts** | `urllib.request.urlopen(req, timeout=30.0)` on all Ollama/Gemini calls | Prevents zombie processes on W540 |
| **Context Window Guard** | Middle-truncate at 12,000 chars before any LLM call | Prevents OOM on local model |
| **Scoped Epistemic Scrubber** | `archive_log(source_type="external")` scrubs; `source_type="internal"` embeds directly | Default is `external` — safe by default |
| **Obsidian Vault Protection** | Dual path validation (`../` & absolute paths), loopback enforcement, Sensitivity Gate | Prevents path traversal and sensitive PII leaking to vault plugins |
